# 1. Модуль интеллектуальной обработки документов (тестовое задание DS)

AI-агент проверки целевого использования льготных сельхоз-кредитов: модуль извлекает
ключевые поля из документов, классифицирует их тип и проверяет, соответствует ли
предмет оплаты условиям льготной программы.

## 1.1 Как запустить

```bash
pip install -r requirements.txt
python demo.py
pytest tests/
```

## 1.2 Технологии и почему

- **Python 3.11+, стандартная библиотека (`re`, `difflib`, `datetime`)**
- **pytest** 
- **LangChain (опционально)** - для LLM-режима `check_subject`, с обязательным
  fallback без ключа

Решение не требует внешних API по умолчанию - код запускается локально без ключей. Регулярные выражения дают быстрый и контролируемый парсинг. Fuzzy-сравнение компенсирует опечатки и OCR-ошибки.

## 1.3 Архитектура

```
src/
  extract.py          # extract(text) -> dict
  classify.py         # classify(text) -> (doc_type, confidence)
  subject_check.py    # check_subject(subject) -> (matches, confidence, reason)
dataset/              # тестовые документы + subjects_test.txt
tests/                # pytest-тесты на все три функции
demo.py               # CLI: прогоняет пайплайн по dataset/ и печатает таблицы
```

Пайплайн для одного документа: `classify(text)` -> определяем тип -> `extract(text)`
-> ключевые поля -> если тип invoice/act, поле `subject` уходит в `check_subject()`
для проверки целевого назначения. При `confidence` ниже порога любой из функций
результат помечается как "требуется ручная проверка".

## 1.4 Компромиссы

- Извлечение суммы: если в тексте несколько сумм (цена за единицу и итог),
  приоритет отдаётся суммам с явным маркером ("Итого", "Сумма", "к оплате").
  Если маркера нет - берётся первое число рядом с валютой, что может быть
  неверно для сложных многотабличных документов.
- Дата в формате `ДД/ММ/ГГ` при коротком годе интерпретируется как `20XX`;
  слэш-формат считается ДД/ММ, а не ММ/ДД (документы русскоязычные).
- `classify` - эвристика на ключевых фразах, а не обученная модель; порог
  `unknown` (0.15) подобран на 7 тестовых файлах и может потребовать
  пересмотра на большем корпусе.
- `check_subject` без LLM использует word-level fuzzy matching только как
  защиту от опечаток (с жёстким порогом схожести), а не для семантического
  сопоставления - если предмет описан совсем нетиповыми словами, потребуется
  либо расширение словаря категорий, либо LLM-режим.

## 1.5 Как проверить

```bash
pytest tests/        # 25 тестов на extract/classify/check_subject
python demo.py       # таблицы: что извлечено/классифицировано по dataset/,
                     # результаты check_subject по subjects_test.txt 
```

## 1.6 Примеры вызовов

Ниже показан пример работы каждой функции в Python-интерпретаторе:

```python

>>> from src.classify import classify
>>> from src.subject_check import check_subject
>>> extract("Поставщик: ООО «ТехАгро», ИНН 7701234567, Сумма: 1 250 000,00 руб.")
{'amount': 1250000.0, 'date': None, 'inn': '7701234567', 'contractor': 'ООО «ТехАгро»', 'subject': None}
>>> classify("Счёт на оплату №12 от 01.03.2025. Банковские реквизиты...")
('invoice', 1.0)
>>> check_subject("аренда офиса")
(False, 1.0, "предмет 'аренда офиса' относится к категории «аренда офиса/помещений» и не связан с сельхоз-деятельностью")
```

# 2. Мотивация участия в проекте:

Мне интересен проект прежде всего как возможность участвовать в создании MVP с нуля: от прототипа до первого работающего агента. Это даёт свободу в выборе подходов, пространство для экспериментов и быструю обратную связь от реальных пользователей.

Моя роль видится в ML-составляющей продукта. Это может быть и разработка пайплайнов извлечения данных, и исследование сложных кейсов, и анализ ошибок для повышения точности. Я открыта к разным задачам в рамках ML-трека, готова быстро осваивать и включаться в новые задачи.

На проект могу выделять 20 часов в неделю в течение как минимум 3 месяцев, с возможностью продления при успешных результатах. В перспективе заинтересована участвовать в развитии продукта на постоянной основе.
