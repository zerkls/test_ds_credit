# Результаты

1. **Извлечение полей (`extract`)** - использованы регулярные выражения с приоритетом для сумм с маркерами («Итого», «к      оплате») и для дат актов/УПД (поиск после «№ … от»).  
   ИНН извлекается очисткой от разделителей с выбором длины 10/12 цифр. Контрагент - по ключевым словам и шаблонам ООО/АО/ИП, кавычки нормализованы.  
   **Результат:** все поля на тестовых документах совпадают с ожидаемой таблицей. Для `scan_ocr_001.txt` ИНН обнулён при отсутствии контрагента - обработана неуверенность.

2. **Классификация (`classify`)** - взвешенное совпадение ключевых фраз. Порог разрыва 0.15 подобран эмпирически: ниже чаще `unknown` (больше ручных проверок), выше - риск ложных срабатываний.  
   **Результат:** все типы определены верно.

3. **Проверка предмета (`check_subject`)** - keyword + fuzzy-сравнение (опционально LLM с fallback). Категории: агрохимия, семена, техника, ГСМ, полевые работы, животноводство, мелиорация.  
   **Результат:** 14 из 14 правильных ответов для чётких случаев, EDGE обработаны обоснованно.

**Слабые места:** OCR-шум (частично компенсируется нормализацией), неисчерпывающий словарь ключевых слов, возможные проблемы с нестандартной записью контрагента.
